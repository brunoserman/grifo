# 0. Record architecture decisions

**Status:** accepted

## Context

Grifo is built by a non-engineer using Claude Code. The code is not the interesting artifact; the reasoning is. A reader should be able to see what was considered, what was rejected, and what was cut, without reading the source.

## Decision

Keep a short ADR per significant decision, written at the moment the decision is made rather than reconstructed later. One file, five paragraphs at most. Reconstructed rationale is confabulation; a decision recorded when the pain was fresh is evidence.

## Consequences

The repository doubles as a portfolio. Many of these ADRs come from things that broke during setup, which is the point: they show real trial and error rather than a plan that went right on the first try.
